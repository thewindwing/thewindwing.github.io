git

2016-05-09
  • git

1. shell

  • shell是一个命令解析器，它接收用户命令，然后调用相应的应用程序
  • shell是Linux中的命令解析器，dos是Windows中的命令解析器如cmd命令窗口
  • 分类：csh ksh bash(Bourne Again shell)等，bash用的最多。

2. bash常见命令

  • 格式 — 命令 [选项] [参数],如 ls -a ./abc
  • pwd 查看当前目录
  • cd 目录名 — 切换目录
  • ls 查看当前目录下的内容
    • ls -a
    • ls -a -s
    • ls -al
  • mkdir 目录名 — 创建目录
  • touch 文件名 — 创建文件
  • cat 文件名 — 查看文件的全部内容
    • tail查看文件的尾部 cat 文件名 tail
  • less 文件名 — 查看文件，有折叠
    • 每次按一次回车多出现一行
    • 按b回退一页
    • 按 f或空格向前翻一页
    • q退出
  • rmdir 空文件夹名 — 删除文件夹，只能删除空文件
  • rm 文件名 — 删除文件
  • rm -rf 目录名 — 删除文件夹
  • mv — 移动文件或重命名
    • mv 要移动的文件名 位置
    • mv 要重命名的文件 新名字
  • cp 文件 位置 — 复制文件
  • tab — 自动补全，连续按两次会将所有的匹配内容全部显示出来
  • >和>> — 重定向
    • >覆盖 echo hello > apache.html
    • >>追加到尾部 echo ‘hello’ >> apache.html
  • | — 管道符，可以将多个命令连接使用，上一次命令的执行结果当成下一次命令的参数
    • grep 匹配内容，一般结合管道符使用
    • ls | grep ‘h’ 匹配h是否在ls结果中

3. vi编辑器

  • vi编辑器时Linux下的标配，如同Windows中的记事本，可以来创建编辑文件，，它是随系统一起安装的文本编辑软件。
  • 三种模式：
    • 命令模式
    • 输入模式
    • 底行模式
    • 输入模式是不能直接进入底行模式的，需要先切换到命令模式

  • vi 文件名 — 打开文件，打开时是命令模式
    ####命令行模式下的命令：
  • 从命令模式切到输入模式的方式：
    • i — 进入编辑模式，当前光标处插入
    • a — 进入编辑模式，当前光标后插入
    • A — 进入编辑模式，光标移动到行尾
    • o — 进入编辑模式，当前行下面插入新行
    • O — 进入编辑模式，当前行上面插入新行
  • ZZ — 保存并退出
  • u — 撤销操作，可多次使用
  • dd — 删除当前行
  • yy — 复制当前行，必须再粘贴才有效
  • p — 粘贴内容
  • ctrl+f — 向前翻页
  • ctrl+b — 向后翻页

    底行模式下的命令：

  • w — 保存
  • w 文件名 — 另存为
  • q — 退出
  • wq — 保存并退出
  • q! — 不保存强制退出
  • e! — 撤销更改，返回到上一次保存的状态
  • set nu — 设置行号

4. git

工作原理

  • 三个工作区
    • 工作目录
    • 暂存区域
    • 本地仓库（git仓库）
  • 四个工作状态
    • 未跟踪—工作目录中
    • 已暂存—暂存区
    • 已提交—本地仓库
    • 已修改—工作目录

使用git管理代码的基本步骤（新项目）

  • 1.初始化用户信息
    • git config –global user.name 用户名
    • git config –global user.email 邮箱
    • git config –list 查看是否配置成功
  • 2.初始化本地仓库
    • git init
  • 3.查看当前仓库的状态
    • git status
  • 4.添加到暂存区
    • git add 文件名
    • git add * 添加全部
    • git add -A 添加全部
  • 5.提交到本地仓库
    • git commit -m 备注记录
    • git commit -amend 修改备注
    • git commit -a -m 备注 — 对于已经add过的文件再次修改提交时，可省略git add直接用这行命令
  • 6.查看提交的本地仓库记录
    • git log
  • 7.上传到远程仓库
    • git push 远程仓库的地址 本地分支的名称:远地分支的名称

回滚操作

  • 撤销工作目录的修改（暂存前）
    • git checkout .撤销全部修改
    • git checkout 文件名撤销指定文件的修改，可以是多个
  • 删除暂存区的文件
    • git rm –cached 文件名
  • 删除本地仓库的几种情况
    • git reset –hard 快照id — 回滚到指定快照id即提交commitid的位置，本地仓库，暂存区和工作目录都回滚了
    • git reset –soft 快照id —回滚到指定快照id，git log可以查看，只是本地仓库回滚
    • git reset –mixed 快照id —默认，回滚到指定的commitid，本地仓库和暂存区回滚
      • 可以简写成git reset 快照id
      • 是git add的你操作
    • HEAD — 指最新的一个快照
      • 上面的所有git reset 中的快照id都可以写成HEAD
      • HEAD^ 表示最新的快照的上一个提交的快照
      • HEAD^^ 表示最新的快照的上上一个提交的快照，依次类推，如git reset –hard HEAD^

分支操作

  • 创建分支
    • git branch 分支名
  • 切换分支
    • git checkout 分支名
  • 创建并切换到分支
    • git checkout -b 分支名称
  • 合并分支
    • git merge 分支名 — 必须在主分支master下合并
  • 查看分支
    • git branch —查看本地分支
    • git branch -r — 查看远程分支
    • git branch -a — 查看所有分支
  • 删除分支
    • git branch –d 分支名 — 删除本地分支，必须合并后才可删除
    • git branch –D 分支名 — 强制删除本地分支
    • git push origin –delete 分支名 —删除远程分支
    • git push origin:分支名 — 删除远程分支

远程仓库

  • 本地创建共享仓库
    • 1.创建一个文件夹以.git结尾—- mkdir 文件名.git
    • 2.进入这个目录— cd 文件名.git
    • 3.初始化为一个共享仓库（裸库）— gitinit –bare
  • 向仓库同步内容
    • git push 远程（共享）仓库地址 本地分支名称:远程分支名称 —- 本地分支名称和远程分支名称相同时可以只写本地分支名称
  • 从远程仓库提取内容
    • git clone 远程仓库地址 项目名称（可选） —- 第一次提取时
    • git pull 远程仓库地址 本地分支名称:远程分支名称 — 当远程仓库有更新 但本地分支没有更新，所以就要再拉取一次用pull

      git中远程仓库机制

  • 给远程仓库起别名
    • git remote add 远程仓库的别名 远程仓库地址
    • 远程仓库的别名一般是origin 即git remote add origin 远程仓库地址
  • 推送到远程仓库
    • git push -u origin master
    • -u 是用来记录路径的，以后再次上传时直接用 git push 就可以了。
  • 查看别名
    • git remote
    • git remote -v — 查看详细信息
    • git remote show 别名 — 查看远程主机的信息
  • 修改别名
    • git remote rename 仓库别名 新别名
  • 删除别名
    • git remote remove 仓库别名

      冲突解决

  • 两个开发同时改到同一文件的同一段内容就会发生冲突
  • 冲突产生后，需要开发者进行协商确认冲突的原因
  • 然后将冲突代码删除重新提交即可

gitignore忽略文件

  • 在项目目录下新建一个文件.gitignore,这是文件名不是后缀，可以在一些编辑器中新建，以防被误认为后缀名；
  • 将一些不希望提交的文件名罗列到这个文件里

    比较差异

  • git difftool — 比较的是工作目录和暂存区的差异
  • git difftool commitid — 比较的是工作目录和特定快照的差异
  • git difftool commitid1 commitid2 — 比较的是两个快照之间的差异
  • git difftool 分支名1 分支名2 — 比较的是两个分支之间的差异

    修复bug

  • 在开发中突然需要紧急修复一个bug，解决办法：
    1. 另外再建一个分支
      • 首先，切换到主分支master
      • 创建新分支修复bug，新分支中并没有原开发分支中的内容
      • 合并分支 bug分支和以前开发的分支合并
    2. 保存现场
      • 保存现场 git stash 保存现场后，工作目录中就没有开发修改的内容
      • 修改bug
      • 恢复现场 git stash pop
      • 修整冲突，合并
