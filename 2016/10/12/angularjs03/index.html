1. jqLite

  • 由于jQuery文件太大，为了方便DOM操作，angularjs在内部提供了一个jQuery的精简版—–叫做jqLite lite简化的
    • 但都是些简化的方法 不拥有完全的功能
  • $(原生JS对象)，将原生JS对象转换成jQuery对象，目的是为了使用jQuery对象下面提供的方法。
  • angular.element(原生JS对象)
    • 相当于jQuery中的$符号—-，将原生JS对象转换成angularjs中的jqLite对象，目的是为了使用jqLite对象下面提供的方法
    • angular.element(document.getElementById(‘box’)); —返回一个伪对象
    • angular.element(document.getElementById(‘box’)).css(‘background’,’red’);

  • 文档：https://code.angularjs.org/1.6.4/docs/api/ng/function/angular.element

3. $scope.$watch的使用

  • $scope.$watch（ ‘val’，function(newValue,oldValue){ } ）—实时监控数据的变化
  • <input type='text' ng-model='val'>
  • 第一个参数是要监控的数据，加引号
  • 第二个参数是回调函数，只要数据发生变化，就会执行回调函数
  • newValue:用户输入的最新的值
  • oldValue：上一次用户输入的内容
  • 页面一上来回调函数首先会执行一次

4.$scope.$apply的使用

  • $scope.$apply() 通知angularjs说： JS中的数据变了，你来帮我更新一下页面。
  • 当通过原生JS将angularjs中的数据做了改变以后，angularjs不知道，所以需要通过 $scope.$apply()方法，通知angularjs更新HTML页面。
  • 只要不使用angularjs提供的JS 都要 $scope.$apply() 通知

5.自定义指令的作用

  • 增强HTML标签的功能
    • 自定义HTML标签
      • 在元素JS中，可以将公共的代码封装成函数
        • 在angularjs中，可以将公共的HTML结构封装成自定义标签
  • 操作DOM
    • 在angularjs中,页面中的DOM与数据是有对应关系的
    • 对数据做增删操作,页面中的DOM会自动增加删除
    • DOM的增删是由angularjs通过指令完成的,比如ng-repeat,ng-if,ng-switch
    • 如果开发人员要对页面中的DOM做额外的操作
    • 需要等待angularjs的操作完成以后进行
    • 但是开发人员是不知道angularjs的操作什么时候完成的
    • 不过angularjs自己是知道的
    • 所以angularjs推荐开发人员也通过指令进行DOM操作
    • 当内部提供的指令执行完成以后,它会去调用开发人员定义的指令
    • 这样就达到了在angularjs操作完DOM以后开发人员再去操作DOM的目的

6.自定义指令的语法

  • 模块对象.directive(‘名字’，[‘$timeout’,function($timeout){
    return { 所有逻辑都在这做 }
    }] )
  • 第一个参数： 自定义指令的名字 自定义指令即自定义标签
  • 第二个参数： 回调函数
  • 注意：要这样写
    • html => my-header
    • js => myHeader
    • angularjs会将模版的内容放到这个标签内。
  • replace:true;//将自定义指令标签删掉，浏览器中浏览页面时就没有这个标签了。
  • transclude:true;//告诉angularjs将自定义标签内部的内容保留，保留到了ng-transclude中

  • 案例：

  • 在angularjs中不推荐在控制器中直接操作DOM，而是通过指令的方式操作

    • link:function(scope,element,attributes){}
    • link函数是angularjs提供给我们的专门写DOM操作的地方
    • element: 指令所在的元素
    • attributes ：指令所在的元素身上的属性集合，是一个对象Attributes
    • scope:向指令的模版区域暴露数据
    • scope:true;使指令两次执行的时候用不同的scope

  • 自定义指令可以用做标签来使用 也可以作为属性使用,同样angularjs会将模版的内容放到属性所在的标签内。

  • 指令两次执行用的是同一个scope，所以会发生覆盖的问题

    • 解决办法：
    • scope:true;使指令两次执行的时候用不同的scope，即有不同的作用域
  • scope到底是什么?
    • 作用域
    • scope：false 指令在默认情况下没有自己的作用域，使用的是控制器的作用域
      • ，即其实就是控制器的$scope
    • scope:true 使指令具有单独的作用域
      • 当前作用域找不到时往外面的控制器的作用域中找
    • scope：false 时 若指令里面也设置同样的数据时，指令的scope和控制器的$scope两个scope是同一个，会显示指令的数据
  • 在angularjs中自定义指令是有类型的
    • 标签指令 E element
      • <my-dir>我是标签指令</my-dir>会将模版的内容放到my-dir中
    • 属性指令 A attributes
      • <div my-dir>我是属性指令</div> 会将模版的内容放到div中
    • 样式指令 C class
      • <div class=my-dir>我是样式指令</div> 不会将模版的内容放到div中
      • 需要配置： restrict:’EACM’
      • restirct: 规定当前指令能够使用的方式
    • 注释指令 M comment
      • 注释指令需要配合replace:true一起使用
      • <!--directive:my-dir 我是注释指令-->
