typeof方式

typeof方式对数组、日期、正则的检测结果都是object。这些并不是我们想要的，我们需要的是Array、Date、RegExp。

Object.prototype.toString.call方式

在typeof的基础上通过Object.prototype.toString.call（）来进一步判断数据类型，这种方式可以做到严谨的校验。

  • 以上两种方法对比：
数据类型 typeof（值都是字符串） Object.prototype.toString.call()
数值 number [object Number]
字符串 string [object String]
布尔值 boolean [object Boolean]
null object [object Null]
undefined object [object Undefined]
new Date() object [object Date]
new Boolean() object [object Boolean]
/^ab$/g object [object RegExp]
Math.pow function [object Function]
函数 function [object Function]
其他普通对象 object [object Object]

浅拷贝

浅拷贝就是只复制对象的一层。即只能改变对象一层的引用了，如果对象属性值还是对象，则复制的是所包含的子对象的引用值。这样当改变副本的成员值时，原对象成员的属性值也会改变。

深拷贝

深拷贝就是一层层的复制对象的所有层级，并且不影响原来的对象内容。

改变浅拷贝问题的解决办法：

  • 深拷贝方法一：person2=JSON.parse(JSON.stringify(person1));

通过先转换为字符串格式，再转为对象格式，这样一转换就全部拷贝过来，即深拷贝。但这种方法虽然简单但是有浏览器兼容问题。

  • 深拷贝方法二：递归

原理： 若想复制一个对象的所有数据且不影响原对象时，需要递归或遍历每个属性并进行判断是否是复杂数据类型，若是就创建一个新的数组或对象，再遍历其成员，若成员还是复杂类型数据则再创建遍历，直到最深层。即通过递归的方法来实现深拷贝。

函数的call、apply和bind方法

2015-06-15

call()方法

  • call用来改变函数内部this指向，并调用该函数
  • 语法：fun.call(thisArg[,arg1[,arg2[,……]]])
  • 参数：
    • 第一个参数用来指定内部this指向，传递一个引用类型或this
      • 如果指定null或undefined则内部this指向window
    • 第二个参数开始指定为函数传递参数，并一一对应传递过去（顺序不能改变），是个参数列表

  • 用途：

    • 借用构造函数实现实例成员的初始化（属性继承）
    • 让伪数组可以借用数组的方法
      [].push.call(this,5)

  • 实例1：

  • 实例2：

var obj={0:’a’,1:’b’,2:’c’,length:3};
//通过call方法改变forEach方法的this指向obj
;[].forEach.call(obj,function(item,index){
console.log(item,index);
})

apply()方法

  • apply方法和call方法作用一样,用来改变函数内部this指向的
  • 语法：fun.apply(thisArg,[argsArray])
  • 第一个参数也是用来指定内部this指向的
  • 区别在于：传参的方式不一样
    • apply第二个参数必须是一个数组，apply会把参数数组展开，然后一一对应的传递给函数
    • 只处理第一个参数数组，后续的都不处理
  • 实例：

bind()方法

  • bind方法也是用来改变函数内部this指向的
  • 语法：fun.bind(thisArg[,arg1[,arg2[,...]]])
  • 和call、apply区别在于：
    • call和apply在改变this指向的同时会直接调用该函数执行
    • 而bind不会调用该函数而是返回一个指定了this环境的新函数
  • 返回值：返回由指定的this值和初始化参数改造的原函数拷贝
  • 参数：
    • 当绑定函数调用时，bing中的参数（第二个开始）将至于实参之前传递给被绑定的方法

  • 主要用途：

    • 改变定时器内部this的指向
    • 改变数组的遍历方法内部this的指向（forEach、every、filter等）
    • 因为这以上两种情况内部的this都指向window

  • 实例：

总结：

  • call和apply特性一样

    • 都是用来调用函数，而且是立即调用
    • 但是可以在调用函数的同时，通过第一个参数指定函数内部this的指向
    • call调用时，参数必须以参数列表的形式传递，即以逗号分隔的方式依次传递即可
    • apply调用时，参数必须是一个数组，然后执行时，会将数组内元素一个一个拿出来与形参一一对应进行传递
    • 如果第一个参数指定了null或undefined，则内部this指向window

  • bind

    • 可以用来指定内部this的指向，然后生成一个改变了this指向的新的函数
    • 和call、apply的最大区别就是：bind不会自调用
    • bind支持传递参数，但传参的方式比较特殊，一共有两个位置可以传递
      • 在bind的同时，以参数列表的形式进行传递
      • 在调用时，以参数列表形式传递

浏览器函数提升中的bug

2015-06-12

if语句中的函数声明方式定义函数可能会产生bug

实例：

调用fn()时，在IE10以下会输出false，因为fn是通过函数声明方式定义的，这些浏览器在解析前会先进行函数提升。

解决办法—用函数表达式来定义函数

但在大多数新得浏览器已解决这个bug，为了兼容这些低版本浏览器最好的方式是，用函数表达式来定义函数。

函数的自定义方式

2015-06-08

函数声明

function f1(){}

  • 最大特性：预解析时具有函数提升
  • 实例：
    console.log(typeof f1);//这里输出的是function，不是number
    var f1=123;
    function f1(){
      console.log(‘hello’);
    }
    console.log(f1); //这里的结果是123
    • 解析：
  • 因为预解析时，同名函数和变量，函数的优先级高，与先后顺序无关。
  • 在执行阶段，因为为函f1重新复制了123，而并没调改函数f1，所以会直接跳过function f1(){}执行下面的代码。

函数表达式

var f1=function(){}

  • 类似于变量赋值，没有函数提升
  • 可以没有名字（匿名函数）

new Function()创建函数

var sum=new Function(‘x’,’y’,’return x+y’);
和下面的sum函数一样
function sum(x,y){
  return x+y;
}
