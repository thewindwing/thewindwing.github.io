函数的call、apply和bind方法

2015-06-15

call()方法

  • call用来改变函数内部this指向，并调用该函数
  • 语法：fun.call(thisArg[,arg1[,arg2[,……]]])
  • 参数：
    • 第一个参数用来指定内部this指向，传递一个引用类型或this
      • 如果指定null或undefined则内部this指向window
    • 第二个参数开始指定为函数传递参数，并一一对应传递过去（顺序不能改变），是个参数列表

  • 用途：

    • 借用构造函数实现实例成员的初始化（属性继承）
    • 让伪数组可以借用数组的方法
      [].push.call(this,5)

  • 实例1：

  • 实例2：

var obj={0:’a’,1:’b’,2:’c’,length:3};
//通过call方法改变forEach方法的this指向obj
;[].forEach.call(obj,function(item,index){
console.log(item,index);
})

apply()方法

  • apply方法和call方法作用一样,用来改变函数内部this指向的
  • 语法：fun.apply(thisArg,[argsArray])
  • 第一个参数也是用来指定内部this指向的
  • 区别在于：传参的方式不一样
    • apply第二个参数必须是一个数组，apply会把参数数组展开，然后一一对应的传递给函数
    • 只处理第一个参数数组，后续的都不处理

bind()方法

  • bind方法也是用来改变函数内部this指向的
  • 语法：fun.bind(thisArg[,arg1[,arg2[,...]]])
  • 和call、apply区别在于：
    • call和apply在改变this指向的同时会直接调用该函数执行
    • 而bind不会调用该函数而是返回一个指定了this环境的新函数
  • 返回值：返回由指定的this值和初始化参数改造的原函数拷贝
  • 参数：
    • 当绑定函数调用时，bing中的参数（第二个开始）将至于实参之前传递给被绑定的方法

  • 主要用途：

    • 改变定时器内部this的指向
    • 改变数组的遍历方法内部this的指向（forEach、every、filter等）
    • 因为这以上两种情况内部的this都指向window

总结：

  • call和apply特性一样

    • 都是用来调用函数，而且是立即调用
    • 但是可以在调用函数的同时，通过第一个参数指定函数内部this的指向
    • call调用时，参数必须以参数列表的形式传递，即以逗号分隔的方式依次传递即可
    • apply调用时，参数必须是一个数组，然后执行时，会将数组内元素一个一个拿出来与形参一一对应进行传递
    • 如果第一个参数指定了null或undefined，则内部this指向window

  • bind

    • 可以用来指定内部this的指向，然后生成一个改变了this指向的新的函数
    • 和call、apply的最大区别就是：bind不会自调用
    • bind支持传递参数，但传参的方式比较特殊，一共有两个位置可以传递
      • 在bind的同时，以参数列表的形式进行传递
      • 在调用时，以参数列表形式传递

函数的自定义方式

2015-06-08

函数声明

function f1(){}

  • 最大特性：预解析时具有函数提升
  • 实例：
    console.log(typeof f1);//这里输出的是function，不是number
    var f1=123;
    function f1(){
      console.log(‘hello’);
    }
    console.log(f1); //这里的结果是123
    • 解析：
  • 因为预解析时，同名函数和变量，函数的优先级高，与先后顺序无关。
  • 在执行阶段，因为为函f1重新复制了123，而并没调改函数f1，所以会直接跳过function f1(){}执行下面的代码。

函数表达式

var f1=function(){}

  • 类似于变量赋值，没有函数提升
  • 可以没有名字（匿名函数）

new Function()创建函数

var sum=new Function(‘x’,’y’,’return x+y’);
和下面的sum函数一样
function sum(x,y){
  return x+y;
}
