函数原型链

2015-06-10
  • 函数原型链

每一个函数都是Function的实例

var f1=new Function();

函数查找机制

所有函数的原型对象都指向Function.prototype，包括Function自己
即：

f1.__proto__===Function.prototype;
Function.__proto__===Function.prototype;

函数的自定义方式

2015-06-08
  • 函数的自定义方式

函数声明

function f1(){}

  • 最大特性：预解析时具有函数提升
  • 实例：
    console.log(typeof f1);//这里输出的是function，不是number
    var f1=123;
    function f1(){
      console.log(‘hello’);
    }
    console.log(f1); //这里的结果是123
    • 解析：
  • 因为预解析时，同名函数和变量，函数的优先级高，与先后顺序无关。
  • 在执行阶段，因为为函f1重新复制了123，而并没调改函数f1，所以会直接跳过function f1(){}执行下面的代码。

函数表达式

var f1=function(){}

  • 类似于变量赋值，没有函数提升
  • 可以没有名字（匿名函数）

new Function()创建函数

var sum=new Function(‘x’,’y’,’return x+y’);
和下面的sum函数一样
function sum(x,y){
  return x+y;
}

构造函数中的继承

2015-05-04
  • 继承

原因：

  • 当两个或多个构造函数具有很多相同的属性和方法时，我们就抽象出一个父类（构造函数）来写公共的属性和方法。

具体做法：

  • 父构造函数：

    • 公共的属性：写到父构造函数内
    • 公共的方法：写到父构造函数的原型对象内

  • 子构造函数：

    • 属性继承：用父.call方法改变this指向来继承
    • 方法继承：
      • 原型对象拷贝继承for-in
      • 利用原型链继承

对象属性的搜索访问机制

2015-04-16
  • 搜索访问机制


原理：

  • 当你访问对象的某个成员时：

    • 现在自己身上找，找到即返回；
    • 找不到，则沿着原型链向上查找，如果在上一个原型对象中找到，则返回；
    • 找不到，则沿着原型链继续查找，直到Object.prototype；
    • 如果还找不到，则沿着Object.prototype.__proto__继续查找；
    • 这时发现Object.prototype.__proto__指针指向null，则停止查找，返回undefined。
