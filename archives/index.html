对象属性的搜索访问机制

2017-08-26

date: 2015-05-04 23:29:59


原理：

  • 当你访问对象的某个成员时：

    • 现在自己身上找，找到即返回；
    • 找不到，则沿着原型链向上查找，如果在上一个原型对象中找到，则返回；
    • 找不到，则沿着原型链继续查找，直到Object.prototype；
    • 如果还找不到，则沿着Object.prototype.__proto__继续查找；
    • 这时发现Object.prototype.__proto__指针指向null，则停止查找，返回undefined。


实例：

function Person(name,age){
this.name=name;
this.age=age;
};
Person.prototype.singing=function(){
console.log(‘you are my super star…’);
};
var child1=new Person(‘lily’,18);
child1.singing();

原型链查找机制

备注：

  • 原型对象–prototype

    • 每一个函数都有一个属性prototype指向一个对象（原型对象），这个对象的所有属性和方法都会被实例继承；

    • 一般每个实例对象的方法都定义在这个对象的原型上面，如

      Array.prototype.push;

      var arr=new Array();

      console.log(arr.push===Array.prototype.push); //true

  • __proto__
    • 每一个对象都有一个__proto__属性（这是一个非标准属性），指向原型对象；
    • 在一些主流浏览器中可以通过对象的__proto__属性访问实例对象的原型对象；

构造函数中的继承

2015-05-04

原因：

  • 当两个或多个构造函数具有很多相同的属性和方法时，我们就抽象出一个父类（构造函数）来写公共的属性和方法。

具体做法：

  • 父构造函数：

    • 公共的属性：写到父构造函数内
    • 公共的方法：写到父构造函数的原型对象内

  • 子构造函数：

    • 属性继承：用父.call方法改变this指向来继承
    • 方法继承：
      • 原型对象拷贝继承
      • 利用原型链继承

实例解析：

function Teacher(name,age){
this.name=name;
this.age=age;
this.speaking=function(){
console.log(‘speaking’);
}
this.teaching=function(){
console.log(‘teaching’);
}
}

function Student(name,age){
this.name=name;
this.age=age;
this.speaking=function(){
console.log(‘speaking’);
}
this.studying=function(){
console.log(‘studying’);
}
}

上面两个构造函数具有一些相同的属性和方法，抽象出一个公共的构造函数

function Person(name, age) {
this.name = name
this.age = age
}
Person.prototype.speaking = function () {
console.log(‘speaking’);
}

Teacher和Student简化为：

function Teacher(name,age){ //属性继承
Person.call(this,name,age);
this.teaching=function(){
console.log(‘teaching’);
}
}
extend(Teacher.prototype,Person.prototype);//方法继承–拷贝继承

function Student(name,age){
Person.call(this,name,age);
this.studying=function(){
console.log(‘studying’);
}
}
extend(Student.prototype,Person.prototype);//方法继承–拷贝继承

拷贝继承封装函数：

function extend(target,source){
for(var key in source){
target[key]=source[key];
}
}

其实实战中最常用的方法继承是原型链继承

Teacher.prototype=new Person();
Study.prototype=new Person();
