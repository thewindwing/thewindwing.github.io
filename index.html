检测数据类型的方式

2015-06-28

typeof方式

typeof方式对数组、日期、正则的检测结果都是object。这些并不是我们想要的，我们需要的是Array、Date、RegExp。

Object.prototype.toString.call方式

在typeof的基础上通过Object.prototype.toString.call（）来进一步判断数据类型，这种方式可以做到严谨的校验。

  • 以上两种方法对比：
数据类型 typeof（值都是字符串） Object.prototype.toString.call()
数值 number [object Number]
字符串 string [object String]
布尔值 boolean [object Boolean]
null object [object Null]
undefined object [object Undefined]
new Date() object [object Date]
new Boolean() object [object Boolean]
/^ab$/g object [object RegExp]
Math.pow function [object Function]
函数 function [object Function]
其他普通对象 object [object Object]

检测构造函数和实例之间关系的方式

  • instanceof 方式

instanceof运算符主要用来判断某个构造函数与实例对象之间的关系。

  • constructor方式

还可以用实例对象的constructor属性值来检测其原型对象

  • 比较

由于我们为某个对象的原型上添加多个方法时，有时会用对象的方式添加，这样自己定义的原型对象会把自带的原型对象覆盖掉，所以需要我们收到添加construor属性，很有可能就会落掉原型对象的constructor属性，所以建议使用instanceof来检测构造函数和实例之间的关系。

对象或数组中的深拷贝和浅拷贝

2015-06-28

浅拷贝

浅拷贝就是只复制对象的一层。即只能改变对象一层的引用了，如果对象属性值还是对象，则复制的是所包含的子对象的引用值。这样当改变副本的成员值时，原对象成员的属性值也会改变。

深拷贝

深拷贝就是一层层的复制对象的所有层级，并且不影响原来的对象内容。

改变浅拷贝问题的解决办法：

  • 深拷贝方法一：person2=JSON.parse(JSON.stringify(person1));

通过先转换为字符串格式，再转为对象格式，这样一转换就全部拷贝过来，即深拷贝。但这种方法虽然简单但是有浏览器兼容问题。

  • 深拷贝方法二：递归

原理： 若想复制一个对象的所有数据且不影响原对象时，需要递归或遍历每个属性并进行判断是否是复杂数据类型，若是就创建一个新的数组或对象，再遍历其成员，若成员还是复杂类型数据则再创建遍历，直到最深层。即通过递归的方法来实现深拷贝。
